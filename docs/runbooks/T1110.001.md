# Password guessing (T1110.001)

A specialisation of T1110 where the adversary throws a dictionary of
passwords at a small number of accounts. SIGMA rule `wazuh-bruteforce`
typically fires on this path — see `services/correlator/app/sigma/rules`.

## Triage

- Is this against a **service account** or a human user? Service
  accounts fail-then-succeed often look like real attacks; check the
  scheduler / cron logs.
- Confirm the source IPs — a single host hammering one user is the
  hot signal.

## Investigation

- Pull the **last successful** auth for the targeted account from
  Wazuh and verify it matches expected geography / time-of-day.
- If the source matches a known internal scanner, mark FP and **also**
  add a Layer-3 suppression so the same alert doesn't fire next week.

## Containment

- Lock the targeted account if there is even a single successful login
  from the bursting source.
- Block the source IP at perimeter; add to the IOC list under
  `services/correlator/app/intel/data/ioc.json` for durable
  enrichment in Phase 2.

## Eradication & lessons

- Force-rotate passwords for any account that authenticated during the
  attack window.
- Recommend MFA / phishing-resistant 2FA on the affected service.
