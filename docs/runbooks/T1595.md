# Active scanning (T1595)

Reconnaissance against your perimeter — the firewall ingestor sees
this as bursts of denied SYN/ACK traffic from a single source. Often
benign (researcher scanners, Shodan); sometimes a precursor to T1190.

## Triage

- Cross-check the source against **known-benign scanners** (Shodan,
  Censys, internal vuln scanners). Those should be Layer-1 whitelisted.
- Determine whether the scan is **port-scanning** (T1046) or
  **vuln-scanning** (T1190 follow-up).

## Investigation

- Was any port answered (`SYN/ACK`)? Tag those services for
  configuration review.
- Did the scanner pivot to actual exploit attempts within the next
  hour? Promote the incident's severity if so.

## Containment

- For known-noisy benign scanners, add a Layer-1 whitelist entry
  rather than a Layer-3 suppression — operator-curated signals beat
  ad-hoc suppressions.

## Eradication & lessons

- Verify perimeter firewall rules match the **actual** services that
  should be exposed (no orphan listeners).
- Consider rate-limit / fail2ban-style controls for unauthenticated
  endpoints.
