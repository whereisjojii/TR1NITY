# Valid accounts (T1078)

The adversary is using legitimate credentials. This is the
"impossible-travel" / "first-time-from-this-country" / "first-time-
from-this-device" signal. Brute-force runbooks pivot here when **any**
member event shows a successful login from the bursting source.

## Triage

- Determine whether the user typically logs in from this geography.
- Was this preceded by a brute-force burst (T1110.x)? If yes, treat
  as confirmed compromise.

## Investigation

- Pull the user's **complete** session timeline for the past 24 h. Are
  there parallel sessions from disjoint geographies?
- Check device posture / MDM enrolment for the source machine.
- If the user is a privileged role, treat as a major incident and
  notify the on-call lead.

## Containment

- Force-logoff the user's active sessions.
- Reset the user's password and rotate any non-MFA secrets they have
  access to (PATs, cloud keys, SSH keys).

## Eradication & lessons

- Audit the past 30 days of activity for the account. Look for
  resource creation, privilege grants, and exfiltration patterns.
- Consider MFA / phishing-resistant 2FA mandate for the role.
