# Phishing (T1566)

Email-based delivery of a payload or credential-harvesting page. The
ingestor surfaces this from the mail-gateway logs (when configured) or
indirectly when a user clicks → T1059 / T1190.

## Triage

- How many recipients? A 1-of-1 message is targeted (T1566.003); a
  blast against many users is opportunistic.
- Pull the URL / attachment hash and run it through your sandbox or
  threat-intel of choice.

## Investigation

- Did anyone click? Any users authenticate to a lookalike domain?
  Pivot to T1078 if so.
- Walk the sender domain — DMARC / SPF status, recent registration
  date, similar past campaigns.

## Containment

- Pull the message from inboxes (zero-hour purge, mail-gateway
  recall).
- Block the sender + URL + attachment hash at the gateway.
- Force password rotation for any user who entered credentials.

## Eradication & lessons

- User awareness micro-training for the targeted population.
- File a takedown request for the lookalike domain.
