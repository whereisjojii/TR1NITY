# Command and scripting interpreter (T1059)

Hostile use of legitimate interpreters: `bash`, `cmd.exe`,
`powershell.exe`, `python`, `osascript`. Wazuh decodes process-creation
events that fan into this technique.

## Triage

- Pull the **command line** verbatim. Encoded payloads (`base64`,
  hex, ROT) are a strong signal.
- Cross-reference parent process: `winword.exe → cmd.exe` is the
  classic phishing → exec signature.

## Investigation

- Decode any encoded payload you see — a sandbox is cheap, an APT is
  not.
- Look for the next 5 minutes: outbound HTTPS to a low-reputation
  domain, file drops, registry runs.

## Containment

- Kill the process tree on the host.
- Isolate the host from the network.

## Eradication & lessons

- Walk persistence mechanisms.
- For Windows hosts, consider enabling AMSI + ScriptBlock logging if
  not already on. PowerShell logging level should be 5 (full).
