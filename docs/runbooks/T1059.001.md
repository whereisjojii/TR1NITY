# PowerShell (T1059.001)

A specialisation of T1059. Living-off-the-land via PowerShell remains
one of the most common Windows post-exploitation tools.

## Triage

- Look for `-EncodedCommand`, `IEX (New-Object Net.WebClient)`,
  `DownloadString`, `Invoke-Expression`, or any of the classic
  Empire / Cobalt Strike one-liners.
- Was the calling process Office (Word/Excel/Outlook) or a web
  worker? Office → PowerShell is rarely benign.

## Investigation

- If `-EncodedCommand`, decode the base64 and inspect — paste into a
  sandbox if tooling allows.
- Pull AMSI events for the same window. AMSI bypasses themselves are
  a strong signal.

## Containment

- Kill the PowerShell tree.
- Disable PowerShell for the affected user via AppLocker / WDAC if
  the role doesn't legitimately need it.

## Eradication & lessons

- Re-enable Constrained Language Mode where compatible.
- Consider PowerShell Remoting allow-listing.
