# Data encrypted for impact / ransomware (T1486)

Files are being encrypted on a host. This runbook applies once the
encryption activity is **observed** — by the time you see the ransom
note, you're well past triage.

## Triage

- Confirm the alert is real (not a backup tool encrypting at rest).
  File-extension changes (`.locked`, `.crypt`, random 8-char) are a
  strong signal.
- Treat as an emergency: invoke your IR plan, page the on-call.

## Investigation

- Identify **patient zero** by walking the timestamps backwards from
  the first encryption event.
- Check shared filesystems: ransomware spreads via SMB / NFS shares.
  Anything currently mounted on patient zero is at risk.

## Containment

- Cut the host from the network at the switch.
- Block the C2 / payment domain at the perimeter.
- For shared storage, snapshot **immediately** and unmount from the
  affected hosts.

## Eradication & lessons

- Restore from backups. Verify backup integrity _before_ connecting
  the restored host to the network.
- Postmortem: how did the operator land? T1190? T1078? T1566?
- Confirm offline backup retention covers the dwell time of the
  attacker — ransomware operators tamper with online backups.
