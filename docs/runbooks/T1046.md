# Network service scanning (T1046)

An adversary maps your **internal** services (not perimeter — that's
T1595). Either an internal compromise (post-T1078 lateral movement) or
an authorized pen-tester. Either way: investigate before suppressing.

## Triage

- Is the source IP an internal asset? Look it up in the asset
  inventory — flag any host that has no business scanning.
- Check `members[*].destination.port` distribution: a wide port range
  in seconds → automated tooling (nmap, masscan).

## Investigation

- Did the source recently authenticate from an unusual location? If
  yes, escalate to T1078.
- Pull process telemetry from the source host: `nmap`, `masscan`,
  `crackmapexec`, `bloodhound`, `evil-winrm` are all hot signals.

## Containment

- If the source is a workstation, isolate it.
- If it's a server, take a memory snapshot before isolating —
  forensics will need it.

## Eradication & lessons

- Walk the host's persistence mechanisms (cron, scheduled tasks,
  services). Reimage if compromise is confirmed.
