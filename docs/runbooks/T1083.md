# File and directory discovery (T1083)

Adversary enumerates filesystems, often via `find`, `dir /s`,
`Get-ChildItem`, `tree`, or by listing UNC shares. SIGMA rules in this
project flag suspicious `find` patterns from web service accounts.

## Triage

- Is this a service account that **never** runs interactive commands?
  Web app users running `find` is almost always a foothold signal.
- Was this preceded by T1190 alerts in the same window?

## Investigation

- Pull the full process tree from the parent shell. Look for the
  initial command that spawned the discovery (e.g. `bash -c` from a
  web worker).
- Check for outbound connections from the same host in the discovery
  window — discovery often precedes exfiltration (T1041).

## Containment

- Snapshot + isolate the host.
- Block outbound traffic to internet-facing IPs from the host.

## Eradication & lessons

- Audit web-app sandboxing: web workers should not have read access
  to non-app paths.
