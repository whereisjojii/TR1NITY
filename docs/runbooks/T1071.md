# Application-layer protocol / C2 (T1071)

Adversary uses HTTP(S), DNS, or other application-layer protocols for
command-and-control. Often surfaces via firewall denies (when blocked)
or via Wazuh alerts on suspicious user agents / unusual destinations.

## Triage

- Are the destinations recognised? `intel_hits` is the first place to
  look.
- Check the user-agent on HTTP traffic — automated tools often have
  default UAs (`python-requests/2.x`, `Mozilla/5.0 (Windows NT 6.0)`).

## Investigation

- Pull the inbound process — what spawned the connection? Match it to
  process telemetry.
- Compute `bytes_out / bytes_in` ratio: heavy out-bound is
  exfiltration; long-lived low-rate is C2 beaconing.

## Containment

- Block the destination at the egress firewall.
- Add the IOC to the bundled file-based provider so future incidents
  light up.
- Isolate the host if a process tree confirms execution.

## Eradication & lessons

- Walk persistence on the source host.
- Consider TLS-inspection at the gateway for outbound HTTPS where
  legally permissible.
